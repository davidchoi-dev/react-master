# React 접근성

React 앱 개발 시 모든 사용자를 고려한 접근성 준수가 필요합니다. 모든 서비스와 디자인의 시작과 끝은 '사용자'이기 때문이죠. 게다가 접근성을 준수하면 보조기기(AT)에서 정보를 해석할 수 있어 보조기기 사용자의 접근성을 향상시킬 수 있으며, 검색 엔진 최적화(SEO)도 향상시킬 수 있습니다. React는 접근성을 갖춘 웹 사이트, 앱을 만들 수 있도록 모든 지원을 하고 있으며, 대부분 표준 HTML 기술이 사용됩니다.

AOA Youtube 채널(접근성 연구반 운영)을 구독하면 접근성을 개선하는 다양한 방법을 시청하실 수 있습니다.

# 헤딩 레벨(Heading Level)

웹 사이트 또는 애플리케이션 구축 시 제목의 계층 구조 형성이 왜 중요하며, React 앱에서 어떻게 구성하면 좋을지에 대해 다룹니다.

# 히든 콘텐츠(Hidden Contents)

화면에서 감추되, 스크린 리더에서 읽혀야 하는 숨김 콘텐츠 처리 방법에 대해 이야기하고 React 앱에서 처리하는 방법에데 대해 다룹니다.

# 접근성 자동 검사 (React-axe)

영상 강의를 시청하며 글을 읽고 실습하면 더욱 효과적입니다.

Deque Labs의 axe-core 테스트 도구를 래핑한 React-axe 라이브러리는 React 앱의 모든 페이지 접근성을 자동으로 검사하여 개발자에게 실시간 피드백을 전달합니다. 피드백을 확인하고 문제를 해결함으로 잠재적인 접근성 문제를 미연에 방지할 수 있습니다. 접근성을 잘 모르는 개발자로 하여금 오류를 해결하게 함으로서 접근성을 교육할 수 있습니다.

react-axe 모듈 패키지 설치

$ npm i react-axe
1

엔트리 파일(index.js) 설정



 

 
 
 



import React from 'react'
import ReactDOM from 'react-dom'
import axe from 'react-axe'

if (process.env.NODE_ENV !== 'production') {
  axe(React, ReactDOM, 1000)
}

ReactDOM.render(<App />, document.getElementById('root'));

# 오류 메시지 한글화

오류 메시지를 한글로 변경하면 발생한 접근성 오류를 이해하기 쉬워 손쉽게 문제를 해결해 접근성 품질을 향상시킬 수 있습니다. 한글화 파일인 axe-core_locale-ko.zip을 다운로드한 후 압축을 풀고 axe-core_locale-ko/ 디렉토리를 프로젝트 src 디렉토리 내부로 이동시킵니다.

axe-core 한글화 ZIP 파일 다운로드

axe-core_locale-ko.zip

엔트리 파일(index.js) 설정




 







import React from 'react'
import ReactDOM from 'react-dom'
import axe from 'react-axe'
import './axe-core_locale-ko'

if (process.env.NODE_ENV !== 'production') {
  axe(React, ReactDOM, 1000)
}

ReactDOM.render(<App />, document.getElementById('root'));
