# React만으로 상태 관리

이 섹션은 Context API + React Hooks을 활용해 Redux와 유사한 상태 관리 방법을 다뤄보는 보너스 챕터입니다. 해당 내용에 관심 있고 도전하고 싶다면? Redux 학습에 앞서 읽고 실습해보세요. 다소... 어렵게 느껴진다면 Redux를 먼저 학습한 후 다시 도전하세요!

# React Hooks의 등장

Front-End 웹 애플리케이션을 구축하기 위한 최고의 라이브러리 중 하나인 React 라이브러리는 지난 몇 년간 많은 변화를 겪었습니다. 그 중 하나가 React 새로운 개념인 훅(Hooks, v16.8+)의 도입입니다. 훅은 React 앱의 컴포넌트를 작성하는 보다 간편하고 편리한 방법을 방법을 제공합니다. 일반적으로 자주 사용되는 훅은 다음과 같습니다.

  • useRef
  • useState
  • useEffect
  • useContext
  • useReducer
  • useCallback

위에 나열된 훅 외에도 React는 다음의 훅을 제공합니다.

훅 설명
useImperativeHandle forwardRef와 함께 사용 (부모에서 자식 컴포넌트 핸들러 명령)
useLayoutEffect useEffect를 먼저 사용 권장. 문제가 있을 경우 활용
useDebugValue 개발자 도구에서 훅의 레이블 표시 설정
useMemo 메모이제이션 값을 반환 (고 비용 계산 방지)

# Hooks이 가져온 변화

React 라이브러리는 성능과 사용성 면에서 매우 인기있지만, 프레임워크의 모든 기능을 제공하지는 않습니다. 예를 들어 컴포넌트 간 데이터 통신을 위해 중앙에서 상태를 관리하는 기능 등은 제공되지 않습니다. 그래서 외부 라이브러리(예: Redux)를 추가로 설치해 통합해 사용합니다. 하지만 React 훅을 제공하면서 React 앱의 상태 관리를 직접 수행할 수 있게 되었습니다. 다시 말해 Redux와 같은 상태 관리 라이브러리를 별도로 설치하지 않고도 React 자체 기능 만으로 상태 관리를 할 수도 있습니다.

# Redux 사용 경험

이미 Redux(A 상태 관리) 라이브러리를 활용하는 프로젝트를 경험한 적이 있다면? React 훅을 사용해 상태 관리하는 방법을 배우는 것이 매우 수월할 것입니다. Redux 아키텍처(설계 방법)의 기본 원리에 대해 리마인드 해봅시다.

Redux 아키텍처가 생소하다면? 다음 챕터로 넘어가 Redux를 학습하는 것이 좋습니다.

# React Redux의 핵심

React + Redux 앱은 핵심 운영 요소는 다음과 같습니다.

  • 스토어(Store) : 컴포넌트와 통신하며 중앙에서 상태 관리 (상태 업데이트 → 컴포넌트 렌더링)
  • 액션 크리에이터(Action Creator) : type, payload 속성을 가진 객체(액션)을 생성 반환하는 함수
  • 리듀서(Reducers) : 전달 받은 액션의 타입을 감지하여 상태를 변경하는 순수 함수

# 4. 액션 생성(Action Creator) 함수

액션(type, payload 속성을 가진 객체)을 생성하는 함수를 작성합니다. 액션 생상 함수는 컴포넌트에서 "할 일 추가(addTodo)", "할 일 완료(doneTodo)" 등의 처리를 리듀서에 요청(dispatch) 할 때 사용됩니다.

export const addTodo = (newTodo) => {
  return {
    type: ADD_TODO,
    payload: newTodo
  }
}

export const doneTodo = (doneTodoID) => {
  return {
    type: DONE_TODO,
    payload: doneTodoID
  }
}

# 5. 리듀서(Reducer) 함수

액션 생성 함수가 반환한 액션(type, payload 속성을 가진 객체)이 요청(dispatch)에 의해 리듀서에 전달되면 리듀서 함수는 액션의 타입(Type)을 확인하여 조건(액션 상수 값과 비교)과 일치하는 처리(할 일 추가 또는 할 일 완료. 혹은 기본 처리)를 수행합니다.

const reducer = (state, action) => {
  switch (action.type) {
    // 할 일 추가 (새로운 상태 반환)
    case ADD_TODO:
      const newTodo = action.payload;
      return {
        todos: [...state.todos, newTodo]
      };
    // 할 일 완료 (새로운 상태 반환)
    case DONE_TODO:
      const doneTodoID = action.payload;
      return {
        todos: state.todos.filter(todo => todo.id !== doneTodoID)
      };
    // 기본 (현재 상태 반환)
    default:
      return state;
  }
};

# 6. 스토어 공급자 래퍼 컴퍼넌트

useReducer 훅은 상태(state), 요청(dispatch)를 반환하는 훅으로 Redux의 리듀서 함수와 유사한 설계 패턴을 제공합니다. 이 훅은 리듀서 함수와 상태(초기 값)을 전달 받습니다. (3번째 인자는 '지연된 초기화'를 수행할 때 값을 전달하지만, 현재 실습에서는 필요하지 않습니다.)

보다 자세한 useReducer 훅 사용법은 공식 문서를 참고하세요.

import React, { createContext, useReducer } from "react";

// 생략

export const StoreProvider = props => {
  const [ state, dispatch ] = useReducer(reducer, initState);
  return (
    <Store.Provider value={{ state, dispatch }}>
      {props.children}
    </Store.Provider>
  );
};

# store.js

지금까지 순조롭게 코드를 작성했다면? src/store/store.js 파일 코드는 다음과 같이 작성되었을 겁니다. 실습에서는 모든 코드를 하나의 파일에서 관리하지만, 스토어가 복잡해지면 각 모듈(액션 상수, 액션 생성 함수, 리듀서 함수 등)을 개별 파일로 분리하여 관리 하는 것이 좋습니다.

    type: ADD_TODO,
    payload: newTodo

export const doneTodo = removeId => {
