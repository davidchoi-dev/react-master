# AppNavigation 컴포넌트

AppNavigation 컴포넌트는 상태, 라이프 사이클 훅을 사용한 클래스 컴포넌트입니다. React 훅을 활용하는 컴포넌트로 변경하는 실습을 진행합니다.

학습 목적으로 여러 방법을 다뤄보는 것일 뿐. 실제 프로젝트에서는 팀 또는 개인이 선호하는 방법을 사용하세요.

  • 클래스 컴포넌트 → 상태, 라이프 사이클 훅
  • 함수형 컴포넌트 → React 훅

# React Hooks 호출

이번 학습에 사용되는 React 훅은 다음과 같습니다. 나열된 각 훅을 클릭하면 상세한 설명을 볼 수 있습니다.

  • useRef
  • useState
  • useEffect
  • useContext
  • useCallback

먼저 사용할 React 훅을 호출하는 구문을 작성합니다.

# useContext 훅

클래스 컴포넌트에서만 사용 가능한 contextType 대신, 함수형 컴포넌트는 useContext 훅을 사용해 컨텍스트로부터 공급된 값(value)을 함수 안에서 사용 가능한 지역 변수로 설정하여 활용할 수 있습니다.

// 컨텍스트
const { navigation: { title, items } } = useContext(AppContext);

# useState 훅

AppNavigation 컴포넌트의 열림/닫힘 상태와 내비게이션 클래스 속성 이름 상태를 useState 훅을 사용해 각각 관리합니다.

// 상태
const [ isOpen, setOpen ] = useState(false);
const [ navClassName, setNavClassName ] = useState('appNavigation');

왜? 이번에는 상태를 통합 관리하지 않죠?

통합 관리하면 편리함도 있지만, 때로는 불편함을 가질 수도 있습니다. 다루고 있는 예제의 경우 시간 차를 두고 hidden 속성, is-active 클래스 속성 추가 여부를 설정해야 하므로 개별 관리가 유용합니다. 통합 관리 유무는 사용 환경에 따라 달라질 수 있습니다.

# useCallback 훅

함수형 컴포넌트에서 사용할 각 핸들러를 정의할 때 useCallback 훅을 사용합니다. (성능 이슈)

# 내비게이션 메뉴 열기 핸들러

메뉴 열림(true) 상태로 변경한 후, 시간 차(100ms)를 두고 내비게이션 클래스 이름에 is-active를 추가하도록 코드를 변경합니다.

// 핸들러
const handleOpenMenu = useCallback(
  (e) => {
    setOpen(true);
    window.setTimeout(() => setNavClassName(navClassName + ' is-active'), 100);
  },
  [ navClassName ]
);

useCallback 훅은 의존성 배열에 등록된 아이템 중 하나가 변경 될 때만 다시 실행됩니다.

# 내비게이션 메뉴 닫기 핸들러

내비게이션 클래스 이름에 is-active를 제거한 후, 시간 차(400ms)를 두고 메뉴 닫힘(false) 상태로 변경되도록 코드를 변경합니다.

const handleCloseMenu = useCallback((e) => {
  setNavClassName('appNavigation');
  window.setTimeout(() => setOpen(false), 400);
}, []);

# 닫기 버튼 포커스 이동 핸들러

closeButton 참조의 current는 실제 DOM 노드를 가리킵니다. 해당 요소로 포커스 이동되도록 코드를 수정합니다.

const handleFocusCloseMenuButton = useCallback((e) => {
  if (e.shiftKey && e.keyCode === 9) {
    e.preventDefault();
    closeButton.current.focus();
  }
}, []);

# useEffect 훅

기존 클래스 컴포넌트에서 사용되었던 shouldComponentUpdate 라이프 사이클 훅을 useCallback 훅으로 대체할 수 있습니다. isOpen 상태가 변경되면 handleCloseMenu 핸들러를 이벤트에 추가하거나, 제거하여 성능 저하를 막습니다.

// 라이프 사이클 훅 대체
useEffect(
  () => {
    // Esc 키 메뉴 닫기 핸들러(포함)
    const handleEscCloseMenu = (e) => {
      e.keyCode === 27 && handleCloseMenu();
    };

    const methodName = isOpen ? 'addEventListener' : 'removeEventListener';
    window[methodName]('keydown', handleEscCloseMenu);
  },
  [ isOpen, handleCloseMenu ]
);

# AppLoading 스타일 컴포넌트

AppLoading 컴포넌트 style 속성에 설정된 스타일 객체 대신, 스타일 컴포넌트 StyledAppLoading로 변경해 활용해봅니다. 이번 스타일 컴포넌트는 전달 받은 props에 따라 동적 변경되도록 코드를 구성해봅니다.

import styled from 'styled-components';

// 스타일 컴포넌트
const StyledAppLoading = styled(AppLoading)`
  position: fixed;
  top: 50%;
  left: 50%;
  width: ${({size}) => size || 120};
  height: ${({size}) => size || 120};
  transform: translate(-50%, -50%);
`;
