# AppHomeLink 컴포넌트

AppHomeLink 컴포넌트는 이전에 다룬 컴포넌트와 달리 컴포넌트 내부적으로 래퍼(Wrapper) 컴포넌트를 가집니다. 그런 이유로 전달 속성(props, wrapperProps)을 처리하는데 다소 복잡함을 가집니다. 외부의 스타일 파일을 통해 관리하는 방법은 크게 다르지 않으므로, 이번 학습은 styled-components 라이브러리를 활용하는 방법을 살펴봅니다.

학습 목적으로 여러 방법을 다뤄보는 것일 뿐. 실제 프로젝트에서는 팀 또는 개인이 선호하는 방법을 사용하세요.

  • CSS 또는 Sass 스타일 파일을 사용해 관리
  • CSS 또는 Sass 모듈을 활용한 관리
  • CSS-in-JS 라이브러리를 활용한 관리

# wrapperProps.as 속성 처리

래퍼 컴포넌트는 전달된 wrapperProps.as 속성 값에 따라 동적으로 변경 처리됩니다. 그런 이유로 스타일 컴포넌트화 할 경우, as 속성을 어떻게 처리해야 할까 고민될 수 있습니다. 하지만 스타일 컴포넌트 또한 as 속성을 전달 받아 처리하므로 설정하고자 하는 요소를 사용해 기본 처리 할 수 있습니다.

# 동적 이미지 경로

별도의 스타일 코드가 아닌, 컴포넌트 자체 스타일 관리 시 주의할 점은 배경 이미지의 url() 값이 동적으로 변경되므로 import문을 사용해 이미지 경로를 가져와 사용해야 한다는 점입니다. 아래 코드에서 강조된 부분을 살펴보세요.


 


















 




import styled from 'styled-components';
import logoUrl from '@assets/ediya-logo.svg';

// 스타일 컴포넌트 (기본 값인 h1으로 처리)
const StyledAppHomeLink = styled.h1`
  margin-top: 0;
  margin-bottom: 0;

  .animate & {
    opacity: 0;
    transform: translateX(-4rem);
    animation: transform-none 0.3s 0.7s cubic-bezier(0, 0, 0.23, 1.43) forwards;
  }

  a {
    box-sizing: content-box;
    display: block;
    width: 152px;
    height: 15px;
    padding: 26px;
    background: url(${logoUrl}) no-repeat center;
    background-size: 152px 15px;
  }
`;

# 불필요해진 className 속성 병합

AppHomeLink 컴포넌트를 스타일 컴포넌트로 변경함에 따라 컴포넌트 내부적으로 코드가 다소 변경이 필요합니다. 먼저 스타일 컴포넌트는 컴포넌트 자체적으로 스타일을 포함하므로 굳이 외부에서 전달된 className 속성 병합에 대해 신경쓸 필요가 없어졌습니다. 하여 관련 설정 부분 코드를 모두 제거해야 합니다. (아래 코드 주석 참고) 그리고 스타일 컴포넌트에 as 속성을 전달하여 동적 렌더링 처리를 하므로 아래 코드의 as 컴포넌트 설정 부분고 같이 코드를 변경해 사용해야 합니다.

변경 과정을 다루는데 있어 복잡함이 느껴진다면? 다양한 사용법을 학습 중이기 때문일 것입니다. 실제 프로젝트에서는 선호하는 방법을 선택해 사용법에 익숙해진다면 복잡함이 줄어들 것입니다.






 


 



















const AppHomeLink = ({
  external,
  children,
  wrapperProps: { 
    as, 
    // className: wrapperClassName 속성 분해 할당 제거
    ...restWrapperProps 
  },
  // className 속성 분해 할당 제거
  ...linkProps
}) => {
  // as 컴포넌트 설정
  const WrapperComponent = as || 'h1';

  // 렌더링
  return (
    <StyledAppHomeLink as={WrapperComponent} {...restWrapperProps}>
      <a
        {...linkProps}
        target={external ? '_blank' : null}
        rel={external ? 'noopener noreferrer' : null}
      >
        {children || <A11yHidden>홈 링크</A11yHidden>}
      </a>
    </StyledAppHomeLink>
  );
};
