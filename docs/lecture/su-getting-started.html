# Semantic UI 시작하기

# 설치

Semantic UI React는 React 컴포넌트를 제공하고 Semantic UI는 테마를 CSS로 제공합니다. React 컴포넌트를 설치한 후, 필요에 따라 태마(Themes)를 선택하여 사용할 수 있습니다.

  • 컴포넌트 설치
  • 테마 설치 및 설정
  • 사용자 정의 테마

# 컴포넌트 설치

Semantic UI React 컴포넌트를 설치하는 명령은 다음과 같습니다.

$ npm i semantic-ui-react
1

태마를 사용하려면 Semantic UI >=2.3.x, Semantic UI React >=0.81.0 버전 이상이 필요합니다.

TypeScript를 사용을 위해 별도로 설치 할 필요가 없습니다. Typing은 Semantic UI React 패키지에 포함되어 있습니다.

# 테마 설치 및 설정

필요하다면? Semantic UI 기본 테마를 다운로드 받아 사용합니다.

$ npm i semantic-ui-css
1

설치 후에는 엔트리(entry) 파일 상단에서 호출하여 테마를 적용할 수 있습니다.

import 'semantic-ui-css/semantic.min.css'
1

테마 CDN을 사용하고 싶다면?

정적 index.html 파일 </head> 위에 다음 코드를 추가합니다.

<link rel="stylesheet" href="//cdn.jsdelivr.net/npm/semantic-ui@2.4.2/dist/semantic.min.css" />
1

# 사용자 정의 테마

기본 테마 대신 직접 디자인 한 테마를 사용하고자 한다면 사용자 정의 테마 문서를 참고하세요.

# 번들링

Semantic UI React는 모든 최신 JavaScript 번들러를 완벽하게 지원됩니다.

# CRA (Create React App)

Semantic UI React는 CRA와 완벽하게 호환 설치 후, 바로 사용 할 수 있습니다.

# Webpack 4

Semantic UI React는 Webpack v4(Tree Shaking 포함)에서 완벽하게 호환됩니다. (semantic-ui-react >= 0.81.2)

앱을 배포(release)하기 전에 프로덕션(production) 모드로 빌드(build) 하여야 합니다. Semantic UI React는 빌드 과정에서 propTypes 제거 등 프로덕션 모드의 여러 최적화(optimizations)가 포함됩니다.
