# AppHeader 컴포넌트

AppHeader 컴포넌트는 상태 또는 사이드 이펙트를 관리하는 컴포넌트가 아닙니다. 그러므로 이번 학습 주제는 스타일 관리 방법입니다. CSS 모듈 관리 방법과 스타일 컴포넌트 활용 방법을 각각 살펴봅니다.

학습 목적으로 여러 방법을 다뤄보는 것일 뿐. 실제 프로젝트에서는 팀 또는 개인이 선호하는 방법을 사용하세요.

# CSS 모듈 활용

AppHeader 컴포넌트에서 호출하는 AppHeader.scss 파일 이름을 AppHeader.module.scss 파일로 변경한 후 다음과 같이 스타일 모듈 파일을 호출하도록 코드를 수정합니다. 기호에 따라 특정 스타일 모듈만 가져와 사용할 수 있습니다. (CSS 또한 동일한 방법 사용)

import { appHeader } from './AppHeader.module.scss';
1

가져온 스타일 모듈을 적용하여 고유한 클래스 속성 이름 값을 설정합니다.

const AppHeader = () => (
  <header className={appHeader}>
    {/* 생략 */}
  </header>
)

CSS 모듈을 사용하는 이유?

CSS 스타일 관리의 가장 어려운 부분은 모든 것을 전역(Global)에서 관리해야 한다는 점입니다. 이름, 속성 등이 동일한 경우, 충돌 문제가 발생하고 우선 순위(cascade)에 따라 스타일이 최종 반영되므로 스타일 관리가 어려운 것이지요. 이러한 문제를 해결하기 위해 등장한 방법 중 하나가 CSS 모듈(Module)입니다. CSS 모듈을 사용하면 고유한 이름을 사용하여 충돌 문제로 인한 고충을 줄일 수 있습니다.

# CSS-in-JS 활용

앞서 살펴본 것처럼 별도의 스타일 파일을 통해 컴포넌트 스타일을 관리하지 않고, 컴포넌트 자체적으로 스타일을 포함하는 방법인 스타일 컴포넌트를 활용하는 방법을 살펴봅니다. 이러한 방법은 컴포넌트를 활용하는 React에 적합한 CSS-in-JS 방법론으로 JS 파일 안에서 CSS를 사용합니다. 기존의 웹 개발 방법인 스타일 시트 묶음 관리 대신, 컴포넌트 레벌로 추상화(모듈)하여 스타일을 관리할 수 있습니다.

All You Need To Know About CSS-in-JS 원문을 번역한 CSS-in-JS에 관해 알아야 할 모든 것을 참고로 읽어보세요. 학습 중인 styled-components 외에도 다른 CSS-in-JS 라이브러리 사용법을 확인할 수 있습니다.

# CSS-in-JS 라이브러리 설치

프로젝트에 styled-components 패키지가 설치되어 있는지 확인한 후

$ npm ls styled-components
1

설치되어 있지 않다면? 아래 명령을 사용해 프로젝트에 패키지를 설치합니다.

$ npm i styled-components
1

# 스타일 문서 → 컴포넌트

별도의 스타일 파일인 AppHeader.scss를 불러오는 대신, AppHeader 컴포넌트 안에서 스타일을 관리하도록 합니다. 라이브러리를 불러와 스타일 코드를 포함하는 스타일 컴포넌트를 만듭니다.

import styled from 'styled-components';

// 스타일 컴포너트
const StyledAppHeader = styled.header`
  position: fixed;
  z-index: 600;
  top: 0;
  left: 0;
  right: 0;
  display: flex;
  justify-content: space-between;
  background: rgba(255, 255, 255, 0.4);
  margin-bottom: 30px;
  border-bottom: 1px solid #e0e0e0;

  .animate & {
    /* 0% */
    opacity: 0;
    transform: translateY(-4rem);
    /* 100% */
    animation: transform-none 0.35s 0.4s ease-out forwards;
  }
`;

/**
 * AppHeader 컴포넌트
 */
const AppHeader = () => {
  return (
    <StyledAppHeader>
      {/* 생략 */}
    </StyledAppHeader>
  )
}

styled-components 사용에 대해

기존 스타일 파일을 변경하는 방법으로 학습한 것이라 다소 불편하게 느낄 수도 있지만, 프로젝트 시작 단계부터 사용한다면 꽤 편리합니다. vscode-styled-components 패키지를 설치하면 스타일 컴포넌트의 스타일 코드 영역에서 코드 컬러링을 사용할 수 있습니다.

다만, Emmet 단축 코드를 활용할 수 없는 점이 다소 아쉽습니다. 이 문제에 대해 psullivan6는 JSX와 CSS-in-JS를 혼용해 사용하기 때문이 아닐까라고 원인을 추론하였습니다. 그는 별도의 sytles.js 파일을 만들어 스타일 컴포너트를 관리, 호출하는 방법을 소개하기도 하였습니다. 다만, 이 방법을 사용하려면 VS Code의 설정을 다음과 같이 변경해야 합니다. 그리고 .jsx와 .js 파일을 나눠 사용해야 합니다.

"emmet.includeLanguages": {
  "javascript": "css"
},
